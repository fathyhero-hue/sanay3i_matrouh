Patch 14 - Admin Override Approval Without ID Card Requirement

الغرض:
- السماح للإدارة باعتماد الصنايعي من لوحة الإدارة حتى لو لم يرفع صورة البطاقة وجه وظهر.
- شرط رفع البطاقة ما زال موجودًا في صفحة التسجيل الجديدة، ولا يتم إلغاؤه من نموذج التسجيل.

الملفات المطلوب نسخها:
1) admin.html
2) api/server.js
3) service-worker.js

اختياريًا إذا أردت نسخة كاملة متناسقة مع Patch 13:
- index.html
- worker.html
- vercel.json
- icons/default-worker-avatar.png

التغييرات:
- إلغاء منع الاعتماد من API /api/admin/workers/:id/identity-review عند عدم وجود صور البطاقة.
- إلغاء منع الموافقة المباشرة من زر الموافقة العادي عند عدم وجود صور البطاقة.
- تحديث رسالة نافذة مراجعة التحقق لتوضيح أن الاعتماد يمكن أن يكون استثناءً إداريًا.
- تحديث PWA cache إلى sanay3i-matrouh-v13.

ملاحظات:
- لا يوجد SQL جديد مطلوب.
- التسجيل الجديد سيظل يرفض الطلب إذا لم يتم رفع وجه وظهر البطاقة.
- صور البطاقة لا تظهر للعملاء.

بعد التركيب:
1) أوقف السيرفر.
2) شغّل npm start.
3) افتح لوحة الإدارة.
4) اضغط Ctrl + F5.
5) جرّب اختيار: تم التحقق والاعتماد.
