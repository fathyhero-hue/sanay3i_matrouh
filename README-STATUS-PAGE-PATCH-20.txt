Patch 20 - صفحة متابعة حالة طلب التسجيل
========================================

الهدف:
إضافة صفحة عامة للصنايعي يستطيع من خلالها متابعة حالة طلب التسجيل بعد الحصول على رقم الطلب.

الرابط الجديد:
/status

الملفات المعدلة / المضافة:
- status.html              صفحة متابعة حالة الطلب الجديدة
- register.html            إضافة زر متابعة حالة الطلب بعد نجاح التسجيل
- api/server.js            إضافة API عام آمن للاستعلام عن حالة الطلب
- service-worker.js        تحديث كاش PWA إلى sanay3i-matrouh-v21
- vercel.json              إضافة Rewrite للرابط /status

الـ API الجديد:
GET /api/registration-status?q=SN-2026-00025
GET /api/registration-status?q=01000000000

البيانات التي تظهر للصنايعي:
- رقم الطلب
- الاسم
- الحرفة
- المنطقة
- تاريخ التسجيل
- حالة الطلب
- سبب الحالة المختصر عند الحاجة فقط

البيانات التي لا تظهر:
- صور البطاقة
- مسارات مستندات البطاقة
- ملاحظات الإدارة الداخلية
- بيانات حساسة أخرى

حالات الطلب:
- بانتظار المراجعة
- تم القبول
- تم القبول - غير نشط حاليًا
- يحتاج تعديل بيانات
- إعادة رفع البطاقة
- مرفوض

طريقة التركيب:
1) انسخ الملفات التالية مكان القديمة في جذر المشروع:
   - register.html
   - service-worker.js
   - vercel.json
   - status.html

2) انسخ ملف السيرفر:
   - api/server.js
   إلى:
   backend/api/server.js

3) اقفل السيرفر وشغله من جديد:
   npm start

4) افتح:
   http://localhost:3000/status

5) اعمل Hard Refresh:
   Ctrl + F5

ملاحظات:
- لا يوجد SQL جديد مطلوب، بشرط أنك شغلت SQL الخاص بـ Patch 19 لإضافة registration_code.
- لو لم يكن registration_code محفوظًا لبعض العمال القدامى، النظام يحاول استخراج الطلب من ID داخل كود مثل SN-2026-00025.
